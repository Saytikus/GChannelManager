# Сборка и интеграция

## Требования

| Компонент | Минимум |
|---|---|
| CMake | 3.16 |
| Компилятор | C++20 (GCC 10+, Clang 11+, MSVC 19.29+) |
| Qt | 6 (Core); Qt5 поддерживается через `find_package(QT NAMES Qt6 Qt5)` |
| Сборщик | Ninja / Make / MSBuild |
| Qt Test | только для `GCHANNELMANAGER_BUILD_TESTS=ON` |

## Опции CMake

```cmake
option(GCHANNELMANAGER_BUILD_EXAMPLES "Build the GChannelManager demo executable" OFF)
option(GCHANNELMANAGER_BUILD_TESTS    "Build the GChannelManager unit tests"      OFF)
```

| Опция | По умолчанию | Эффект |
|---|---|---|
| `GCHANNELMANAGER_BUILD_EXAMPLES` | `OFF` | Собирает `examples/GChannelManagerDemo` — loopback-демо с потерями и повторами |
| `GCHANNELMANAGER_BUILD_TESTS` | `OFF` | Собирает `tests/tst_SimpleFrameCodec` и `tests/tst_Gateway`, регистрирует их в `ctest` |

## Сборка из командной строки

```sh
# чистая разработческая сборка с тестами и демо
cmake -S . -B build/Desktop-Debug \
    -DCMAKE_BUILD_TYPE=Debug \
    -DGCHANNELMANAGER_BUILD_TESTS=ON \
    -DGCHANNELMANAGER_BUILD_EXAMPLES=ON

cmake --build build/Desktop-Debug

# запуск тестов
( cd build/Desktop-Debug && LD_LIBRARY_PATH="$PWD" ctest --output-on-failure )
```

## Сборка через Qt Creator

`CMakeLists.txt` обнаруживается стандартным `Open Project…`. По умолчанию Qt Creator кладёт сборку в `build/Desktop-Debug/`. Опции `GCHANNELMANAGER_BUILD_*` доступны в `Projects → Build → Initial CMake Configuration` или через окно `Initial Parameters`.

## Использование как зависимости

### Вариант 1: `add_subdirectory` (моно-репозиторий, vendored)

Простейший способ: положить `GChannelManager` в подкаталог и подключить.

```cmake
# в вашем CMakeLists.txt
add_subdirectory(third_party/GChannelManager)

add_executable(myapp src/main.cpp)
target_link_libraries(myapp PRIVATE GChannelManager)
```

`target_include_directories` библиотеки имеет `PUBLIC` область, так что заголовки доступны автоматически:

```cpp
#include <GChannelManager/Gateway.h>
#include <GChannelManager/SimpleFrameCodec.h>
```

### Вариант 2: `find_package` (установленный пакет)

Библиотека поставляет install/export-правила, поэтому её можно установить и подключать через `find_package`:

```cmake
find_package(GChannelManager REQUIRED)

add_executable(myapp src/main.cpp)
target_link_libraries(myapp PRIVATE GChannelManager::GChannelManager)
```

Как получить пакет — см. [Установка](#установка).

## Структура артефактов

После сборки получаются:

| Артефакт | Где |
|---|---|
| `libGChannelManager.so` (`.dll`/`.dylib`) | `build/Desktop-Debug/` |
| `GChannelManagerDemo` (если `BUILD_EXAMPLES=ON`) | `build/Desktop-Debug/examples/` |
| `tst_SimpleFrameCodec`, `tst_Gateway` (если `BUILD_TESTS=ON`) | `build/Desktop-Debug/tests/` |

## Макрос экспорта `GCHANNELMANAGER_EXPORT`

```cpp
// include/GChannelManager/GChannelManager_global.h
#if defined(GCHANNELMANAGER_LIBRARY)
#define GCHANNELMANAGER_EXPORT Q_DECL_EXPORT
#else
#define GCHANNELMANAGER_EXPORT Q_DECL_IMPORT
#endif
```

Все экспортируемые классы помечены этим макросом: `Gateway`, `GatewayRequest`, `ITransport`, `SimpleFrameCodec`. При сборке самой `.so` определяется `GCHANNELMANAGER_LIBRARY` (через `target_compile_definitions(... PRIVATE ...)`), при подключении из внешнего проекта макрос вычисляется как `Q_DECL_IMPORT`.

> [!WARNING] Windows
> На MSVC символы по умолчанию **не** экспортируются. Если добавляете новый публичный класс — обязательно поставьте `GCHANNELMANAGER_EXPORT` перед его именем, иначе он не будет виден из приложения, использующего DLL.

## Минимальный `CMakeLists.txt` потребителя

```cmake
cmake_minimum_required(VERSION 3.16)
project(MyApp LANGUAGES CXX)

set(CMAKE_AUTOMOC ON)
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

find_package(QT NAMES Qt6 Qt5 REQUIRED COMPONENTS Core)
find_package(Qt${QT_VERSION_MAJOR} REQUIRED COMPONENTS Core)

add_subdirectory(external/GChannelManager)

add_executable(myapp src/main.cpp)
target_link_libraries(myapp PRIVATE
    GChannelManager
    Qt${QT_VERSION_MAJOR}::Core
)
```

## Установка

Библиотека устанавливает версионированную `.so`, публичные заголовки и CMake-пакет
(targets + config + version), так что `find_package` у потребителя работает «из коробки»:

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
cmake --install build --prefix /your/prefix
```

Под префиксом появляется:

| Артефакт | Расположение |
|---|---|
| `libGChannelManager.so.0.1.0` (+ симлинки `.so.0`, `.so`) | `lib/` |
| публичные заголовки | `include/GChannelManager/` |
| `GChannelManagerConfig.cmake`, `…ConfigVersion.cmake`, `…Targets.cmake` | `lib/cmake/GChannelManager/` |

Потребитель указывает CMake на префикс (`-DCMAKE_PREFIX_PATH=/your/prefix`) и
использует `find_package(GChannelManager REQUIRED)`, как показано выше. Экспортируемая
цель — `GChannelManager::GChannelManager`; её config заново находит `Qt::Core` как
зависимость. Версия пакета совпадает с версией проекта (совместимость `SameMajorVersion`).
